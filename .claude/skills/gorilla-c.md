# Gorilla Commit Skill 🦍🍌🚬

GORILLA COMMIT AND PUSH ALL CHANGES. VELOCITY IS KING.

## WHAT GORILLA DO

1. **SEE ALL CHANGES** - `git status` TO FIND ALL BANANA
2. **GROUP CHANGES** - SPLIT INTO LOGICAL COMMITS BY FEATURE/AREA
3. **COMMIT GORILLA STYLE** - SHORT TITLES. ALL CAPS. EMOJIS. 🦍🍌🚬
4. **PUSH FAST** - NO WAIT. SHIP OR DIE.

## COMMIT MESSAGE FORMAT

```
<TYPE>: <SHORT DESCRIPTION IN CAPS>. <EMOJI>

Co-Authored-By: Claude Opus 4.5 <noreply@anthropic.com>
```

### TYPES (GORILLA STYLE)
- `SMASH` = fix/refactor
- `BUILD` = new feature
- `SHARPEN` = improve/optimize
- `CLEAN` = cleanup/delete

### EMOJIS BY AREA
- 🦍 = core/general
- 🍌 = UI/frontend
- 🚬 = config/tools
- ⚡ = performance

## EXAMPLE COMMITS

```
SMASH: FIX BAD BANANA IN CACHE. 🦍
BUILD: ADD GORILLA MODE SKILL. 🚬
SHARPEN: MAKE EDITOR FAST LIKE APE. ⚡
CLEAN: REMOVE ROTTEN CODE. 🍌
```

## GORILLA WORKFLOW

```bash
# 1. SEE CHANGES
git status

# 2. GROUP BY AREA (examples)
# - apps/web/src/fs/* -> FS COMMIT
# - apps/web/src/editor/* -> EDITOR COMMIT
# - packages/* -> PACKAGE COMMIT
# - .claude/* -> TOOLS COMMIT

# 3. COMMIT EACH GROUP
git add <files>
git commit -m "TYPE: DESCRIPTION. EMOJI"

# 4. PUSH ALL
git push
```

## RULES

- NO LONG MESSAGES. SHORT. PUNCHY. 🦍
- ALWAYS ADD Co-Authored-By
- PUSH IMMEDIATELY AFTER ALL COMMITS
- IF FAIL, SMASH HARDER 🍌🚬
